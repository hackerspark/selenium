JUnit

What is Junit?

JUnit is a unit testing framework for the Java programming language. JUnit has been important in the development of test-driven development, and is one of a family of unit testing frameworks.

The JUnit framework provides a ready-made test harness for executing unit tests as well as an API for reporting the success or failure status of a test.

What are the features of Junit?

The following are the features of JUnit:

  • JUnit is an open source framework which is used for writing test cases and executing them.
  • JUnit is very simple and easy to use and requires less time.
  • It provides Annotation support to identify the test methods.
  • It also provides Assertions for comparing actual and expected results.
  • Integration with popular tools Ant, maven and popular IDEs like Eclipse, Jbuilder.

What are the available JUnit annotations?

The following table gives an overview of the most important available annotations:

NoAnnotationDescription
1@Test The @Test annotation identifies a method as a test method.
2@Test (expected = Exception.class)Fails if the method does not throw the named exception.
3@Test(timeout=100)Fails if the method takes longer than 100 milliseconds.
4@Before This method is executed before each test. It is used to prepare the test environment (e.g., read input data, initialize the class).
5@AfterThis method is executed after each test. It is used to clean up the test environment (e.g., delete temporary data, restore defaults). It can also save memory by cleaning up expensive memory structures.
6@BeforeClassThis method is executed once, before the start of all tests. It is used to perform time intensive activities, for example, to connect to a database. Methods marked with this annotation need to be defined as static to work with JUnit.
7@AfterClassThis method is executed once, after all tests have been finished. It is used to perform clean-up activities, for example, to disconnect from a database. Methods annotated with this annotation need to be defined as static to work with JUnit.
8@IgnoreIgnores the test method.

What are the available JUnit assertions?

The following table gives an overview of the assertions. Parameters in [] brackets are optional.

NoAssertionDescription
1fail(String)Let the method fail. Might be used to check that a certain part of the code is not reached or to have a failing test before the test code is implemented. The String parameter is optional.
2assertTrue([message], boolean condition)Checks that the Boolean condition is true.
3assertFalse([message], boolean condition)Checks that the Boolean condition is false.
4assertEquals([String message], expected, actual)Tests that two values are the same.
5assertEquals([String message], expected, actual, tolerance)Test that float or double values match. The tolerance is the number of decimals which must be the same.
6assertNull([message], object)Checks that the object is null.
7assertNotNull([message], object)Checks that the object is not null.
8assertSame([String], expected, actual)Checks that both variables refer to the same object.
9assertNotSame([String], expected, actual)Checks that both variables refer to different objects.

What is Test Suite?

A Test Suite is a grouping unit test cases together and running it. In JUnit, both @RunWith and @Suite annotations are used to run the suite test.

What is @After and @AfterClass?

@After annotation:

Syntax:

@After
public void myTest()

This method should execute after each test and used for cleaning up the test and temporary data to avoid memory issues.

@AfterClass annotation:

Syntax:

@AfterClass
public static void myTest()

This method should execute at the end, once all the tests are finished. Method should be declared static and executes only a single time. Mostly used for closing the database connection.

What is @Before and @BeforeClass?

@Before annotation:

Syntax:

@Before
public void myTest()

This method should execute before each test. Such methods are generally used for initialization before performing a actual test in test environment.

@BeforeClass annotation:

Syntax:

@BeforeClass
public static void myTest ()

This method should execute before all the tests. It executes only once. Method should be declared static. Mostly used for database connectivity tasks before execution of any of the test.

How do you ignore a test in JUnit?

@Ignore annotation is used to skip or ignore a test case/ test method.

public class JUnit_Ignore{

 @Ignore("Ignore this test")
 @Test
 public void TestOne(){
   System.out.println("This test will not be executed");
 }

 @Test
 public void TestTwo(){
   System.out.println("This test will be executed");
 }
}

What are annotations?

Annotations are like meta-tags that you can add to your code and apply them to methods or in class. The annotation in JUnit gives us information about test methods, which methods are going to run before & after test methods, which methods run before & after all the methods, which methods or class will be ignore during execution.

What is expected annotation in JUnit?

We use "expected" with @Test annotation and specify the type of exceptions that are expected to be thrown when executing the test methods.

The below example which throws an exception called “ArithmeticException” when dividing two numbers with denominator value as 0. The test will pass since we handled the exception by “expected” annotation.

@Test(expected=ArithmeticException.class)
public void dividedByZeroEx(){
 int e=1/0;
}

When you run the test without “(expected = ArithmeticException.class)”, then it will throw the exception “java.lang.ArithmeticException: / by zero”. And the test will fail.

@Test
public void dividedByZeroExample2(){

What is Timeout Test in JUnit?

Some tests will take longer time then what we expect. To handle these tests by using “timeout” method and proceed to execute further tests.

@Test(timeout=5000) 
public void timeOutExample(){
 System.out.println(“hello world”);
}

The above test will wait for 5000 mille seconds.

What is Parameterized test in JUnit?

Parameterized tests are possible in JUnit and they provide us the liberty of passing parameters into the test classes.

@RunWith(Parameterized.class)For making a class parameterized.
@ParametersParameterized class must have a static method for generating and returning a collection of array, this method should be marked with @Parameters annotation.

What is the use of @Rule annotation?

@Rule annotation is used for creating objects, which later can be used in test methods.

What is JUnitCore class?

This class is mainly responsible for executing tests. The org.junit.runner.JUnitCore class has a runClasses() method, which allows us to run one or more test classes. As an output we get a Result (org.junit.runner.Result) object, which we use to filter out the test information.
